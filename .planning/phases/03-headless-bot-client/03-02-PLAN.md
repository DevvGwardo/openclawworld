1. **Add action methods to `bot/BotClient.js`:**

   **`move(toGridPos)` method:**
   - Guard: throw Error "Cannot move: not in a room" if `this.position` is null
   - Guard: throw Error "Invalid target: must be [x, y] array" if `toGridPos` is not an array of length 2
   - Emit `"move"` with `(this.position, toGridPos)` — server expects `(from, to)`
   - Optimistically update `this.position = toGridPos` (research confirms destination tracking is correct — server sets character.position to from, but path ends at to)
   - Critical: Do NOT filter own playerMove events to update position — the optimistic update is sufficient for Phase 3. Filtering would require tracking path animation timing which is unnecessary for a headless client.

   **`say(message)` method:**
   - Guard: throw Error "Cannot say: not connected or not in a room" if no socket or no id
   - Guard: throw Error "Message must be a non-empty string" if message is not a string or is empty
   - Emit `"chatMessage"` with `(message)` — server broadcasts as `playerChatMessage` with `{id, message}`

   **`emote(emoteName)` method:**
   - Guard: throw Error "Cannot emote: not in a room" if no socket or no room
   - Emit `"emote:play"` with `(emoteName)` — server validates against ALLOWED_EMOTES (dance, wave, sit, nod)
   - Do NOT validate emote names client-side — let the server be the authority (server silently ignores invalid emotes)

   **`dance()` method:**
   - Guard: throw Error "Cannot dance: not in a room" if no socket or no room
   - Emit `"dance"` — legacy handler, kept for backward compat (decision from Phase 2)

2. **Create `bot/index.js` entry point:**

   Script that demonstrates a complete bot lifecycle:
   ```
   - Import BotClient from ./BotClient.js
   - Parse SERVER_URL from env (default "http://localhost:3000")
   - Create BotClient with { serverUrl, name: "ClawBot" } (use default avatar)
   - Log "Connecting to {serverUrl}..."
   - await bot.connect() — log available rooms
   - Pick first room: welcomeData.rooms[0]
   - await bot.join(room.id) — log "Joined room {name}, position: {position}"
   - bot.say("Hello! I'm ClawBot, a headless bot.") — log "Sent greeting"
   - Wait 2 seconds (setTimeout wrapped in promise)
   - bot.move([5, 5]) — log "Moving to [5, 5]"
   - Wait 2 seconds
   - bot.emote("wave") — log "Waving"
   - Wait 2 seconds
   - Log "Demo complete. Bot will stay connected. Press Ctrl+C to disconnect."
   - Listen for SIGINT: on signal, call bot.disconnect(), log "Bot disconnected", process.exit(0)
   ```

   Use top-level await (ESM supports it). Wrap in try/catch with meaningful error messages (e.g., "Is the game server running on {serverUrl}?").

   Also add a `bot.on("chatMessage", ...)` listener that logs incoming chat messages from other players, to prove bidirectional communication.
