  <files>bot/GatewayClient.js</files>
  <action>
Modify GatewayClient.js to add reconnection with exponential backoff and ping/pong heartbeat monitoring.

**Constructor additions:**
- `_reconnectAttempt = 0`
- `_maxReconnectAttempts = 10`
- `_alive = false`
- `_heartbeatInterval = null`
- `_autoReconnect = true` (can be disabled for clean shutdown)
- Backoff config: `{ initial: 1000, max: 30000, factor: 2, jitter: true }`

**Reconnection logic -- `_scheduleReconnect()`:**
- If `_autoReconnect === false`, do not reconnect (user called disconnect intentionally)
- If `_reconnectAttempt >= _maxReconnectAttempts`, emit `'reconnectFailed'` and return
- Calculate delay: `min(initial * factor^attempt, max)` with jitter (add 0-20% random)
- Increment `_reconnectAttempt`
- Emit `'reconnecting'` with `{ attempt: _reconnectAttempt, delay }`
- `setTimeout(() => this.connect(), delay + jitter)`

**Heartbeat -- start after hello-ok (in connected handler):**
- Set `_alive = true`
- Listen for `pong` on ws: `this._ws.on('pong', () => { this._alive = true; })`
- Start interval (15s default, matching Gateway tick interval):
  ```
  this._heartbeatInterval = setInterval(() => {
    if (!this._alive) {
      this._ws.terminate(); // forces close event -> triggers reconnect
      return;
    }
    this._alive = false;
    this._ws.ping();
  }, 15000);
  ```

**On close handler update:**
- Clear heartbeat interval if running
- Reject all pending requests with reconnection error
- If close code !== 1000 (not intentional) AND `_autoReconnect`, call `_scheduleReconnect()`
- On successful reconnect (hello-ok received), reset `_reconnectAttempt = 0`

**`_rejectAllPending(error)`:**
- Iterate `_pending` map, clear each timer, call reject(error), then clear the map

**`disconnect()` update:**
- Set `_autoReconnect = false` BEFORE closing ws (prevents reconnect on intentional close)
- Clear heartbeat interval
- Close ws with code 1000
- Reject all pending
- Reset state

**`connect()` update:**
- Set `_autoReconnect = true` (re-enable for fresh connections)
- Clean up previous ws if exists (remove listeners, terminate if still open)
- Then proceed with normal connection setup
- Return promise that resolves on hello-ok (same as Plan 01, but now also works for reconnections)

Important: The connect() promise from a reconnect should NOT reject the original caller. Use a separate internal mechanism -- emit 'connected' event for reconnections, only the first connect() call returns a promise.

  <action>
Create `bot/gateway-test.js` -- a standalone script that exercises the full Gateway flow against a live Gateway. This is a manual integration test (not automated unit test) since it requires a running Gateway.

**Environment variables required:**
- `OPENCLAW_GATEWAY_URL` -- WebSocket URL of the Gateway
- `OPENCLAW_GATEWAY_TOKEN` -- auth token for the Gateway

**Script flow:**

```javascript
import { GatewayClient } from './GatewayClient.js';

const url = process.env.OPENCLAW_GATEWAY_URL;
const token = process.env.OPENCLAW_GATEWAY_TOKEN;

if (!url || !token) {
  console.error('Required env vars: OPENCLAW_GATEWAY_URL, OPENCLAW_GATEWAY_TOKEN');
  process.exit(1);
}

const gw = new GatewayClient({ url, token });

// Log events for visibility
gw.on('connected', (hello) => {
  console.log('[Gateway] Connected! Tick interval:', hello.tick?.interval || 'unknown');
});
gw.on('reconnecting', ({ attempt, delay }) => {
  console.log(`[Gateway] Reconnecting (attempt ${attempt}, delay ${delay}ms)...`);
});
gw.on('disconnected', () => {
  console.log('[Gateway] Disconnected');
});
gw.on('error', (err) => {
  console.error('[Gateway] Error:', err.message);
});

try {
  // Test 1: Connect and authenticate
  console.log('--- Test 1: Connect and authenticate ---');
  console.log(`Connecting to ${url}...`);
  await gw.connect();
  console.log('PASS: Connected and authenticated via challenge handshake');

  // Test 2: Send agent prompt and receive response
  console.log('\n--- Test 2: Agent RPC (hardcoded prompt) ---');
  const prompt = 'You are a bot in a multiplayer game. You see a player nearby. Respond with a JSON object: {"action":"say","message":"Hello there!"}';
  console.log('Sending prompt...');
  const result = await gw.invokeAgent(prompt, { timeoutMs: 30000 });
  console.log('Agent response:', JSON.stringify(result, null, 2));
  console.log('PASS: Received agent response');

  // Test 3: Verify response is parseable (basic check)
  // The actual response structure depends on Gateway -- log it for inspection
  console.log('\n--- Test 3: Response validation ---');
  if (result && typeof result === 'object') {
    console.log('PASS: Response is a valid object');
  } else {
    console.log('WARN: Response is not an object -- inspect manually');
  }

  console.log('\n--- All tests passed ---');
  gw.disconnect();
  process.exit(0);

} catch (err) {
  console.error('FAIL:', err.message);
  gw.disconnect();
  process.exit(1);
}
```
