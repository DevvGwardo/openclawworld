2. Add `.device-keys.json` to .gitignore (append to existing file). This file stores the Ed25519 keypair and device token -- must never be committed.

3. Create `bot/DeviceIdentity.js` with two exported functions:

**`loadOrCreateIdentity(path)`**
- If file at `path` exists, read and parse JSON, reconstruct crypto key objects from PEM strings using `crypto.createPublicKey()` and `crypto.createPrivateKey()`. Return `{ publicKey, privateKey, fingerprint, deviceToken }`.
- If file does not exist, generate Ed25519 keypair via `crypto.generateKeyPairSync('ed25519')`. Compute fingerprint as SHA-256 hex hash of the public key's DER (SPKI) export. Save JSON with PEM-encoded keys, fingerprint, and `deviceToken: null`. Return the identity object.

**`signChallenge(nonce, privateKey)`**
- Decode nonce from base64 to Buffer: `Buffer.from(nonce, 'base64')`
- Sign with `crypto.sign(null, nonceBuffer, privateKey)` -- MUST use null algorithm (Ed25519 does not work with createSign)
- Return signature as base64 string

**`saveDeviceToken(path, identity)`**
- Write updated identity JSON to disk (called after receiving deviceToken from hello-ok)

Use `node:crypto` and `node:fs` -- no external dependencies for this module. ESM with named exports.
  </action>

  </verify>

<task type="auto">
  <name>Task 2: GatewayClient with auth handshake and agent RPC</name>
  <files>bot/GatewayClient.js</files>
  <action>
Create `bot/GatewayClient.js` -- a class extending EventEmitter that manages the WebSocket connection to the OpenClaw Gateway.

**Constructor(`options`)**
- `options.url` -- Gateway WebSocket URL (default from `process.env.OPENCLAW_GATEWAY_URL`)
- `options.token` -- auth token (default from `process.env.OPENCLAW_GATEWAY_TOKEN`)
- `options.identityPath` -- path to .device-keys.json (default `new URL('.device-keys.json', import.meta.url).pathname`)
- Initialize: `_state = 'disconnected'`, `_pending = new Map()`, `_queue = []`, `_nextId = 1`
- Load identity via `loadOrCreateIdentity(identityPath)`

**`connect()`**
- Create WebSocket via `new WebSocket(url)`
- On `open`: set `_state = 'awaiting_challenge'` (do NOT send anything -- wait for server challenge)
- On `message`: route to `_handleMessage(data)`
- On `close(code, reason)`: set `_state = 'disconnected'`, reject all pending requests, emit `'disconnected'` event. (Reconnection added in Plan 02.)
- On `error`: emit `'error'` event
- Return a Promise that resolves when `hello-ok` is received (connected state), rejects on close/error before auth completes

**`_handleMessage(raw)`**
- Parse JSON from raw data (call `.toString()` on raw first -- ws delivers Buffers)
- **State `awaiting_challenge`**: Expect `{ type: 'event', event: 'connect.challenge', payload: { nonce, ts } }`. Call `_respondToChallenge(payload)`.
- **State `awaiting_hello`**: Expect response to connect request. If `msg.type === 'res' && msg.id === '0'`:
  - If `msg.ok`: set `_state = 'connected'`, save deviceToken if present (`msg.payload.auth?.deviceToken`), emit `'connected'`, flush queue, resolve connect promise.
  - If `!msg.ok`: reject connect promise with error message.
- **State `connected`**: Handle `res` (route to pending map) and `event` (emit on EventEmitter).

**`_respondToChallenge({ nonce, ts })`**
- Sign nonce via `signChallenge(nonce, identity.privateKey)`
- Export public key as PEM
- Send connect frame:
  ```json
  {
    "type": "req",
    "id": "0",
    "method": "connect",
    "params": {
      "protocol": 3,
      "client": { "name": "openclawworld-bot", "version": "0.1.0" },
      "role": "node",
      "auth": {
        "token": "<env token or undefined>",
        "deviceToken": "<persisted device token or undefined>"
      },
      "device": {
        "id": "<fingerprint>",
        "publicKey": "<PEM>",
        "signature": "<base64 sig>",
        "signedAt": Date.now(),
        "nonce": "<original nonce string>"
      }
    }
  }
  ```
- Set `_state = 'awaiting_hello'`

**`request(method, params = {})`**
- If `_state !== 'connected'`, push to `_queue` array and return promise (resolved when flushed)
- Generate string ID via `String(this._nextId++)`
- For side-effecting methods, caller should include `idempotencyKey` in params
- Send `{ type: 'req', id, method, params }`
- Store `{ resolve, reject, timer }` in `_pending` map
- Timer: 30s timeout, rejects with timeout error, deletes from map
- Return promise

**`_handleResponse(msg)`**
- Look up `msg.id` in `_pending`
- If found: clear timer, delete from map, resolve with `msg.payload` if `msg.ok`, reject with `msg.error` otherwise

**`_flushQueue()`**
- For each queued item, re-call the internal send logic

**`invokeAgent(prompt, options = {})`**
- Generate `idempotencyKey` via `crypto.randomUUID()`
- Step 1: `const { runId } = await this.request('agent', { message: prompt, idempotencyKey, ...options })`
- Step 2: `const result = await this.request('agent.wait', { runId, timeoutMs: options.timeoutMs || 60000 })`
- Return result (expected: `{ status, startedAt, endedAt, error?, ... }` -- the actual response payload from the LLM)

**`disconnect()`**
- If ws exists and open, close with code 1000
- Set `_state = 'disconnected'`
- Reject all pending
- Clear heartbeat interval if any

**`get connected()`** -- returns `_state === 'connected'`

Use ESM. Import `WebSocket` from `ws`, import `{ loadOrCreateIdentity, signChallenge, saveDeviceToken }` from `./DeviceIdentity.js`, import `crypto` from `node:crypto`, import `EventEmitter` from `node:events`.
