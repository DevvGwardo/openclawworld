<task type="auto">
  <name>Task 2: Create perception module</name>
  <files>bot/perception.js</files>
  <action>
Create `bot/perception.js` that exports a `PerceptionModule` class:

**Constructor:** `new PerceptionModule(botClient, options)`
- `botClient` is a BotClient instance (reads `.id`, `.position`, `.characters`, `.name`)
- `options.radius` -- perception radius in grid units (default 6, Chebyshev distance)
- `options.chatHistoryMs` -- chat history window in ms (default 60000)
- `options.maxOwnActions` -- max own actions to remember (default 5)
- Internal state: `this._chatHistory = []` (array of `{ id, name, message, timestamp }`), `this._ownActions = []` (array of `{ type, params, timestamp }`)

**Methods:**

`onChatMessage({ id, message })` -- call from external event handler. Looks up character name from botClient.characters. Pushes to `_chatHistory`. Auto-prunes entries older than `chatHistoryMs`.

`recordOwnAction(action)` -- records an action `{ type, ...params }` with timestamp into `_ownActions`. Keeps only last `maxOwnActions`.

`snapshot()` -- returns a structured object:
```js
{
  self: { id, name, position },
  nearbyPlayers: [{ id, name, position, isBot, distance }],  // within radius, sorted by distance
  recentChat: [{ name, message, secsAgo }],  // pruned to chatHistoryMs
  ownRecentActions: [{ type, params, secsAgo }],
  timestamp: Date.now()
}
```

Distance calculation: Chebyshev distance `max(|x1-x2|, |y1-y2|)` where positions are `[x, y]`.

Exclude the bot itself from nearbyPlayers.

For character name lookup: find character in `botClient.characters` by id, use `character.session?.name` or `character.name` or fall back to `"Player-" + id.slice(0,4)`.

`serialize(snapshot)` -- converts snapshot to compact text string:
```
[You] ClawBot at [3,7]
[Nearby] Alice at [4,8] (dist 1), Bob at [6,5] (dist 3, bot)
[Chat] (12s ago) Alice: Hey there! | (5s ago) Bob: Welcome
[Your recent] move [4,8] (30s ago), say "Hello" (15s ago)
[Time] 2026-01-31T12:00:00Z
```

Rules:
- If no nearby players: `[Nearby] Nobody nearby`
- If no chat: `[Chat] No recent messages`
- If no own actions: `[Your recent] None`
- Truncate chat messages to 80 chars
- Use pipe `|` separator between chat entries (compact)
- Include `(bot)` marker for bot characters in nearby list

<task type="auto">
  <name>Task 3: Create rate limiter module</name>
  <files>bot/rateLimiter.js</files>
  <action>
Create `bot/rateLimiter.js` that exports `createRateLimiter(options)`:

Uses `limiter` package's `RateLimiter` (or `TokenBucket` depending on API -- check limiter v3 API).

**Note on limiter v3 API:** In limiter ^3.0, use `import { RateLimiter } from 'limiter'`. The constructor takes `{ tokensPerInterval, interval, fireImmediately }`. Use `tryRemoveTokens(1)` for sync check and `removeTokens(1)` for async wait.

Configuration:
- `options.burst` -- max tokens (default 3)
- `options.sustained` -- tokens refilled per second (default 1)

Returns an object with:
- `tryConsume()` -- returns `true` if a token was available and consumed, `false` if rate limited. Uses sync `tryRemoveTokens(1)`.
- `async waitForToken()` -- waits until a token is available. Uses async `removeTokens(1)`. Returns once token consumed.
- `getTokensRemaining()` -- returns current token count (for logging).

The limiter v3 `RateLimiter` constructor: `new RateLimiter({ tokensPerInterval: sustained, interval: 'second', fireImmediately: true })`. Note: limiter v3 uses `tokensPerInterval` for refill rate. For burst, we start with `burst` tokens. Since limiter auto-starts at `tokensPerInterval`, we may need to manually add extra tokens or use a wrapper.

**Simpler approach:** If limiter API doesn't cleanly support burst != sustained, implement a minimal token bucket directly:
- `tokens` starts at `burst` (3)
- Every second, add `sustained` (1) tokens, capped at `burst`
- `tryConsume()`: if tokens > 0, decrement and return true; else false
- `waitForToken()`: poll every 100ms or use setTimeout until token available
- Use `setInterval(1000)` for refill

Pick whichever approach is cleaner. The custom bucket is ~30 lines and avoids API uncertainty.
  </action>
  <verify>
```bash
node -e "
import { createRateLimiter } from './bot/rateLimiter.js';
const rl = createRateLimiter({ burst: 3, sustained: 1 });
console.log('t1:', rl.tryConsume()); // true (3 -> 2)
console.log('t2:', rl.tryConsume()); // true (2 -> 1)
console.log('t3:', rl.tryConsume()); // true (1 -> 0)
console.log('t4:', rl.tryConsume()); // false (0, rate limited)
console.log('remaining:', rl.getTokensRemaining()); // 0
// Wait for refill
setTimeout(() => {
  console.log('after 1s:', rl.tryConsume()); // true (refilled 1)
  rl.destroy(); // cleanup interval
  console.log('OK');
}, 1100);
"
```
First 3 calls return true, 4th returns false, after 1s returns true again.
  </verify>
  <done>Token bucket rate limiter with burst 3 / sustained 1 per second, sync tryConsume, async waitForToken, and cleanup destroy method.</done>
