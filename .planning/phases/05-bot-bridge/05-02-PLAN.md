  <action>
Create `bot/actions.js` with:

**1. Zod Schema -- `ActionSchema`**

Use `z.discriminatedUnion("type", [...])` with four action types:

```js
const MoveAction = z.object({
  type: z.literal("move"),
  target: z.array(z.number().int().min(0).max(99)).length(2),
});

const SayAction = z.object({
  type: z.literal("say"),
  message: z.string().min(1).max(200),
});

const EmoteAction = z.object({
  type: z.literal("emote"),
  name: z.enum(["wave", "dance", "sit", "nod"]),
});

const LookAction = z.object({
  type: z.literal("look"),
  target: z.string().min(1),  // player name or id to face
});
```

Export `ActionSchema` as the discriminated union.

**2. `parseAction(raw)` function**

- `raw` can be a string (JSON to parse) or an object (already parsed)
- If string, try `JSON.parse(raw)`. On failure, return `{ ok: false, error: "Invalid JSON" }`.
- If the parsed value is an array, take the first element (multi-action LLM responses -- take first).
- Run through `ActionSchema.safeParse()`.
- On success: `{ ok: true, action: result.data }`
- On failure: `{ ok: false, error: result.error.issues[0]?.message ?? "Validation failed" }`

**3. `executeAction(action, botClient, logger)` function**

Dispatches a validated action to the BotClient:

- `move`: `botClient.move(action.target)` -- log `{ type: "move", target: action.target }`
- `say`: `botClient.say(action.message)` -- log `{ type: "say", message: action.message }`
- `emote`: `botClient.emote(action.name)` -- log `{ type: "emote", name: action.name }`
- `look`: No server-side support yet. Log `{ type: "look", target: action.target }` at debug level with message "look action logged (no server support)". This is a no-op for v1.

Wrap each call in try/catch. On error, log at warn level and return `{ ok: false, error: err.message }`. On success, return `{ ok: true, type: action.type }`.

The `logger` parameter is a pino child logger (or any object with `.info()`, `.warn()`, `.debug()` methods). If not provided, use a no-op fallback.

<task type="auto">
  <name>Task 2: Create idle patrol controller</name>
  <files>bot/idle.js</files>
  <action>
Create `bot/idle.js` that exports an `IdleController` class:

**Constructor:** `new IdleController(botClient, options)`
- `botClient` -- BotClient instance (reads `.position`, calls `.move()`)
- `options.mapWidth` -- grid width (default 15, from base repo's room size)
- `options.mapHeight` -- grid height (default 15)
- `options.patrolIntervalMs` -- time between waypoint changes if stuck (default 5000)
- `options.arrivalThreshold` -- how close to consider "arrived" (default 1, Chebyshev distance)
- Internal state: `this._currentWaypoint = null`, `this._patrolTimer = null`, `this._active = false`

**Methods:**

`start()` -- activates idle patrol. Sets `_active = true`. Picks first waypoint and starts patrol timer.

`stop()` -- stops idle patrol. Clears timer, sets `_active = false`.

`tick()` -- called on each loop iteration (by BotBridge). If not active, return. Check if bot has arrived at current waypoint (Chebyshev distance <= arrivalThreshold). If arrived or no waypoint, pick a new random waypoint and issue `botClient.move(waypoint)`. If not arrived and patrol timer hasn't expired, do nothing (bot is still walking).

`interrupt()` -- called when LLM response arrives. Stops current patrol, sets `_currentWaypoint = null`. Does NOT set `_active = false` (patrol resumes after LLM action).

`_pickWaypoint()` -- returns a random `[x, y]` where x in [1, mapWidth-2] and y in [1, mapHeight-2] (avoid edges). Must be at least 3 units from current position (so the bot actually moves visibly). If can't find one in 5 tries, just pick any valid position.

`get isActive()` -- returns `_active`

The patrol timer re-triggers `tick()` every `patrolIntervalMs` when active, in case the bot gets stuck (move command ignored, no path found, etc.).
  </action>
  <verify>
```bash

import { IdleController } from './bot/idle.js';
// Mock BotClient
const moves = [];
const mock = { position: [7,7], move(pos) { this.position = pos; moves.push(pos); } };
const idle = new IdleController(mock, { mapWidth: 15, mapHeight: 15, patrolIntervalMs: 500 });

// Simulate arrival

idle.tick(); // should pick new waypoint
console.log('moves after arrival:', moves.length); // 2
idle.interrupt(); // should clear waypoint
console.log('waypoint after interrupt:', idle._currentWaypoint); // null
idle.stop();
console.log('active:', idle.isActive); // false
console.log('OK');
"
```
